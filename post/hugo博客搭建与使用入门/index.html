Hugo博客搭建与使用入门

Wed, Apr 1, 2020

一、Hugo二进制文件下载与环境配置

  1. 下载地址： https://github.com/gohugoio/hugo/releases

由于已经是二进制文件，故不依赖于 go 语言环境。

  1. 解压文件，配置 bin 目录

创建文件夹，E:\hugo\bin

将解压后的 hugo.exe 文件拷贝到 bin 目录。

  1. 配置环境变量

将 hugo.exe 路径添加到 path 目录。

  1. 查看是否安装成功

使用 cmd 或 git push，输入 hugo version。

（推荐 git 之后操作均在 git ）

输出类似：

Hugo Static Site Generator v0.68.3-157669A0 windows/amd64 BuildDate: 2020-03-24T12:04:36Z

即下安装成功。

二、创建博客站点与下载主题

  1. 创建博客站点 cd 到 hugo 目录

$ hugo new site myblog

执行后，在 hugo 目录下就会生成 myblog 的站点文件夹，文件目录结构为：

archetypes　(存放default.md，头文件格式，每次新建文章默认显示的头部信息在此修改)

content 　　 (存放博客文章，markdown格式文件)

data (存放自定义或者导入的模板)

layouts (存放网站的数据模板)

static (存放图片、css、js等静态资源)

themes (存放主题文件，每个主题都是一个独立的文件夹)

config.toml (网站配置文件)

  1. 下载主题

在 theme 目录下

$ git clone https://github.com/spf13/hyde.git

(可在https://themes.gohugo.io选择其他主题下载)

修改 myblog 目录下的 config.tomi 文件，对 theme 进行修改：

theme = “hyde”

保存即可。

三、查看本地服务器

在 myblog目录下，输入

$ hugo server

git终端显示

Web Server is available at http://localhost:1313/ (bind address 127.0.0.1)

复制（推荐右键 copy ）该网址到浏览器即可预览。

复制时若使用 ctrl+c 则会终止显示。

四、创建文章

myblog 目录下，输入

$ hugo new post/第一次写hugo博客.md

则会在 content/post 下生成 第一次写hugo博客.md 文件，打开即可编辑内容，

注：.md文件里 draft 要改为 false，这样才可预览和部署到 github。

五、github部署

  1. 登录github账号

  2. create a new reposity reposity name 必须是 ownerName.github.io

修改 myblog 目录下的 config.toml 文件

将 baseURL 换成新建立仓库的地址：

baseURL = “https://ownerName.github.io/"

注：ownerName 是你自己的用户名

  1. 生成 public 文件夹

myblog 目录下，输入

$ hugo

则会 根据 myblog\content\post 里的 .md文件 在 myblog 目录下生成 public 文件夹，每次更新.md 均需使用该命令 更新 public 文件夹

  1. 初始化 git 仓库，并关联 github 仓库

在 public 目录下，输入

$ git init

即可初始化

在 public 目录下，输入

$ git remote add origin https://github.com/ownerName/ownerName.github.io.git

即将本地仓库链接到 github 远程仓库

六、写博客

步骤：

1）新建 .md 文件 在 myblog 目录下：

$ hugo new post/my.md

编辑内容，修改 draft 为 flase，保存

2）本地预览

在 myblog 目录下：

$ hugo server

即可在本地浏览器打开博客

在vs保存后，会在网页实时显示，方便修改

3）生成 public 文件

ctrl+c 取消预览，切换到 myblog 目录下：

$ hugo

4）部署到 github

在 publlc 目录下

添加

$ git add .

提交

$ git commit -m “第一次修改”

推送（会要求输入 github 用户名和密码）

$ git push -u origin master

成功后即可访问：

ownerName.github.io

奥里给！
